Approximate Computing in CFS

The Linux CFS scheduler calls calc_load() every 5 seconds to update system load averages. This research replaces the 3-cycle IMUL with 1-cycle approximations to reduce scheduler overhead on high-core-count servers.

Three Approximation Strategies

Each variant offers a different trade-off between CPU cycles, memory, and error bounds.

Bit-Shift
O(1). Zero memory. Best speed. Systematic drift.
LUT Lookup
O(1) lookup. 2KB L1 cache. Exact α. Pure quantization error.
Horner Poly
O(d). Best for tunable precision. No systematic drift at high m.

EEVDF Scheduling (CFS+)

Earliest Eligible Virtual Deadline First merges latency and period concepts into a unified scheduling class replacing CFS.

CFS: vtime
vtime += delta / weight
fair_clock tracks virtual time per task.
EEVDF: lag + eligibility
ve = lag + (vr - start) / eligibility
eligibility = sqrt(sqrt(w)), 2× faster wakeups.
EEVDF Core Algorithm
/* linux/kernel/sched/deadline.c — EEVDF pick_next() */
static struct task_struct pick_next_eevdf(struct rq *rq)
{
    struct task_struct *min = NULL;
    u64 min_ve = ~0ULL;

    /* Δ: eligibility = min(latency, period / sqrt(w)) */
    list_for_each_entry(tsk, rq->cfs_tasks, se.run_list) {
        u64 eligibility = calc_eligible(tsk);
        u64 ve = tsk->se.vrun + eligibility;
        if (ve < min_ve) {
            min_ve = ve;
            min = tsk;
        }
    }
    return min;
}

Kernel Implementation

C patches ready for kernel/sched/loadavg.c integration.

/* existing kernel — loadavg.c (reference) */
static unsigned long calc_load(unsigned long load, unsigned long exp, unsigned long active)
{
    unsigned long newload;

    newload = load * exp + active * (FIXED_1 - exp);
    if (active >= load)
        newload += FIXED_1 - 1;

    return newload / FIXED_1;
}

/* variant 1 — bit-shift patch */
#define APPROX_SHIFT_K  4

static unsigned long calc_load_bitshift(unsigned long load, unsigned long active)
{
    unsigned long newload;
    newload = (load - (load >> APPROX_SHIFT_K)) + (active >> APPROX_SHIFT_K);
    return max(0UL, newload);
}
